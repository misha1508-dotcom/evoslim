# План разработки проекта "Цифровой штаб для фитнеса (Digital Twin)" (Evoslim)

Проект `Evoslim` представляет собой систему трекинга здоровья и тренировок, где Telegram выступает основным интерфейсом (включая Telegram Mini App), а Python-бэкенд содержит логику ИИ-агентов (Тренер, Нутрициолог, Researcher) с прямым доступом к базе данных PostgreSQL.

## 1. Цель проекта и Архитектура
Создать персонального ИИ-фитнес-ассистента, который полностью автономен, общается строго профессионально и защищен от галлюцинаций. 
**Архитектурное решение:** Полный отказ от сторонних фреймворков агентов (OpenClaw) в пользу написания ИИ-логики (FSM + LangGraph/Anthropic API) прямо внутри нашего Python-бэкенда. Это обеспечит:
1. Максимальный контроль над системными промптами (Hard Guardrails для здоровья).
2. Прямой, быстрый доступ ИИ к SQL-базе тренировок без прокладок.
3. Полную независимость и легкость деплоя всего проекта (бэкенд + БД + фронт) на одном сервере.

## 2. Что мы ДЕЛАЕМ сейчас (MVP - Фаза 1)

### 🗄️ База данных (PostgreSQL)
Создание структуры и таблиц `users`, `workouts`, `health_metrics`, `medical_history`. Настройка миграций Alembic.

### ⚙️ Backend (Python / FastAPI)
- Telegram Webhook для первичной обработки сообщений.
- **FSM (Конечный автомат):** Жесткая маршрутизация пользователя (напр. "в процессе тренировки", "ждет анализа YouTube", "обычный чат").
- **Нативные ИИ-Агенты:** Подключение Anthropic API. Разработка системных промптов, которые на лету подтягивают противопоказания из БД.
- REST API эндпоинты для сохранения данных из Mini App.

### 🖥️ Frontend (Next.js)
- **Telegram Mini App (TMA):** Ультра-простой мобильный UI для ввода веса, повторений и RIR прямо во время подхода.
- **Web Dashboard:** Защищенная зона для врачей/тренеров с графиками веса, сна и кнопкой экспорта всего профиля здоровья в Markdown.

## 3. Что мы ПОКА НЕ ДЕЛАЕМ (Отложено на Фазу 2)

Чтобы запустить ядро системы быстрее, мы временно откладываем:
- **Интеграцию с умными часами по API:** Пока метрики утреннего пульса/сна/стресса вводятся текстом или вытягиваются из Apple Health/Google Fit скриптами пользователя вручную раз в день. Автоматический вебхук браслетов — во вторую очередь.
- **AI Vision для фото:** Бот может просить загрузить фото прогресса и складывать их в базу, но оценивать их (ставить теги на мышцы) нейросеть пока не будет.
- **Researcher-агент для PubMed:** Самостоятельный поиск научных статей ботом отложен. Пока он анализирует только то, что ты ему скинешь (например, ссылки на YouTube).
- **Сложный парсинг сырого `.vcf` файла генетики:** Загружается один раз вручную в виде текстовой выжимки.

## 4. Рекомендованные фичи для внедрения (Идеи на будущее)

1.  **AI Voice Coaching (TTS):** Раз Мини-апп открыт на телефоне во время тренировки, можно добавить голосовые оповещения (через ElevenLabs/TTS) в наушники: *"Отдых закончен. Подход на грудь. В прошлый раз сделал 80кг на 10. Если RIR был 0, не повышай вес"*. Это создает эффект реального присутствия тренера.
2.  **Анализ техники по видео:** В будущем, если ты скинешь кружок с подходом, агент (передав его в Gemini 1.5 Pro / Claude 3.5 Sonnet) сможет указать на ошибку (например, "отрывается таз от скамьи").
3.  **Умный трекинг продуктовых корзин:** Интеграция с чеками магазинов или API доставок для автоматического подсчета нутриентов купленных продуктов (вместо ручного взвешивания каждого приема пищи).
4.  **Синдикация с Apple Health (через Shortcuts):** Сделать iOS Shortcut, который каждое утро по будильнику сам собирает данные о сне и пульсе из Apple Health и отправляет их HTTP-запросом на наш бэкенд без твоего участия.
